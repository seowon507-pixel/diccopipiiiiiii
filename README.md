# 우리마동 알림

내 주변의 실시간 제보, 커뮤니티, 채팅을 지도와 함께 제공하는 React/Vite 애플리케이션입니다.

## 로컬 실행

Node.js 22.12 이상을 권장합니다.

```bash
npm ci
cp .env.example .env.local
npm run dev
```

`.env.local`에 다음 공개 클라이언트 설정을 입력합니다.

- `VITE_KAKAO_MAP_KEY`: 카카오 지도 JavaScript 키
- `VITE_SUPABASE_URL`: Supabase 프로젝트 URL
- `VITE_SUPABASE_ANON_KEY`: Supabase anon 키

설정이 없거나 잘못된 경우 앱은 빈 화면 대신 연결 설정 오류를 표시합니다. 개발 중 샘플 게시글이 필요할 때만 `VITE_USE_DUMMY_DATA=true`를 명시적으로 설정할 수 있습니다. 실제 위치를 대신하는 개발 좌표도 `VITE_DEV_LOCATION_OVERRIDE=true`를 설정한 경우에만 사용됩니다.

성심교정 중심 전체 기능 데모를 사용하려면 `.env.local`에서 두 값을 함께 켭니다.

```bash
VITE_USE_DUMMY_DATA=true
VITE_DEV_LOCATION_OVERRIDE=true
```

더미 모드에는 성심교정 반경 500m 게시글 12개, 전국 게시글 8개, 빈 핀 4개, 댓글·답글·이모지 반응·반경 채팅과 생성 사진 4장이 포함됩니다. 실제 Supabase Auth를 호출하지 않고 `songsim_demo` 샘플 사용자로 바로 진입합니다. 작성·수정·삭제·확인·추천·신고·사진 첨부도 브라우저 메모리에서 동작하며 새로고침하면 초기 샘플 상태로 돌아갑니다. 원격 Supabase에는 어떤 데이터도 쓰지 않습니다. 모든 샘플 글과 댓글·채팅은 화면에서 샘플임을 구분할 수 있게 표시됩니다.

아직 아래 보안 migration을 적용하지 않은 기존 로컬 Supabase로 기능을 점검해야 할 때는 개발 환경에서만 `VITE_ALLOW_LEGACY_BACKEND=true`를 사용할 수 있습니다. 이 모드는 버전형 RPC가 없다는 `PGRST202` 응답에만 기존 API로 전환하며, 권한·검증·네트워크 오류에서는 전환하지 않습니다. 구형 API의 보안·원자성 한계가 있으므로 배포용 기능이 아니며 production build에서는 항상 비활성화됩니다.

## kimiv 기능 개선 내역

2026년 7월 기능 재검사와 실제 사용자 흐름 테스트를 바탕으로 다음을 개선했습니다.

### v10 성심교정 중심 전체 기능 더미데이터

- 가톨릭대학교 성심교정 중심 좌표(37.48678, 126.80139)를 개발 위치 override로 사용합니다.
- 성심교정 반경 500m에 9개 카테고리를 모두 포함한 게시글 12개와 빈 핀 4개를 배치했습니다.
- 부산·제주·서울·대전·광주·대구·강릉·전주에 전국 샘플 게시글 8개를 추가했습니다.
- 생성형 이미지로 제작한 캠퍼스 보행로·역곡 상권 대기줄·부산 해변 행사·제주 돌담길 사진 4장을 포함했습니다.
- 만료형 게시글은 앱을 시작한 시점을 기준으로 상대 시각을 계산해 항상 지도와 피드에서 테스트할 수 있습니다.
- 댓글·한 단계 답글·5종 이모지 반응·신고 누적 숨김·반경 채팅 데이터가 함께 제공됩니다.
- 더미 모드에서도 글·핀·댓글·채팅의 생성과 수정·삭제, 추천·현황 확인, 사진 첨부가 메모리에서 실제 UI 흐름대로 동작합니다.
- 모든 샘플 데이터는 명시적으로 표시되며 원격 Supabase에 쓰지 않고 새로고침 시 초기 상태로 복원됩니다.
- 더미 모드에서는 실제 회원가입·세션·프로필 API를 호출하지 않고 `songsim_demo` 로컬 샘플 계정으로 바로 진입합니다.

### v9 로그인·신고 관리자 보안

- 로그인 전에는 위치 권한, 게시글 조회, Realtime 구독을 시작하지 않도록 인증 경계를 앞당겼습니다.
- 초기 세션 조회 실패가 빈 화면으로 이어지지 않고 로그인 화면의 오류 안내로 복구됩니다.
- 회원가입 인증 링크와 이메일 로그인 링크가 현재 앱 주소로 돌아오도록 redirect URL을 명시했습니다.
- 로그인 화면의 모든 입력에 label을 연결하고 회원가입/로그인 및 로그인 방법 탭에 방향키·Home·End 키보드 조작을 추가했습니다.
- 원격에만 존재하던 `profiles`, 사용자명 중복확인/저장 RPC를 마이그레이션으로 재현 가능하게 만들었습니다.
- 신고는 브라우저가 보내는 임의 식별값 대신 서버의 `auth.uid()`로 계정당 한 번만 집계합니다.
- `admin`/`moderator` 역할은 비공개 DB 테이블에서 관리하며 사용자가 수정할 수 있는 `user_metadata`를 권한 판단에 사용하지 않습니다.
- 관리자에게만 안전한 신고 큐를 제공하고 숨김·복구·기각을 원자적으로 처리하며 모든 조치를 감사 로그에 기록합니다.
- 신고자 hash와 사용자 UUID는 관리자 화면에도 반환하지 않습니다.
- 공개 게시글 이미지는 계속 표시하되 `post-images` 객체 목록 열람은 클라이언트 역할에서 차단합니다.
- 원격 적용이 필요한 절차는 [`SUPABASE_OWNER_ACTIONS.md`](SUPABASE_OWNER_ACTIONS.md)에 분리했습니다.

### v8 디자인 실험실 — 독립 디자이너 UI 시안 5종 (Ui-kimi 보관 브랜치 기록)

> 현재 `v5` 브랜치는 다섯 시안을 함께 제공하지 않으며, 최종 선택된 Friendly Mobile 계열 UI 하나만 사용합니다. 아래 내용은 `Ui-kimi` 브랜치에 보관한 비교 실험 기록입니다.

- 서로 다른 전문성을 가진 디자인 서브에이전트 5명이 각자 독립적으로 브리프를 만들었습니다. 철학을 화면 문구나 장식적 비유로 노출하지 않고, 정보 구조와 조작 방식에 반영했습니다.
- `03 Civic Grid`는 스위스 공공정보 디자인을 바탕으로 4px 그리드, 각진 선형 목록, 명확한 번호 체계와 코발트 포인트를 사용합니다.
- `04 Korean Editorial`은 한국어 타이포그래피를 중심에 두고 비대칭 편집 그리드, 큰 제목과 엇갈린 콘텐츠 리듬을 사용합니다.
- `05 Friendly Mobile`은 한 손 조작과 48px 터치 영역을 우선하며 따뜻한 색, 선택적 라운드, 모바일 네이티브 하단 탐색을 사용합니다.
- `06 Spatial Desk`는 전문 공간정보 도구처럼 지도·우측 패널·좌측 레일을 분리하고 고밀도 목록과 컴팩트한 제어를 사용합니다.
- `07 Clear Stack`은 고대비·저인지부하·200% 확대 대응을 우선하며 장식을 최소화한 단일 열과 하나의 명확한 주 행동을 사용합니다.
- 다섯 시안은 색만 바뀌는 스킨이 아니라 타이포, 화면 골격, 컴포넌트 밀도, 내비게이션과 모바일 동작이 서로 다릅니다. 기존 기능과 `02 골목 게시판` 복귀 경로는 유지됩니다.
- 320×568에서 다섯 시안 모두 가로 넘침 없음, 선택 시안 자동 노출, 네 개 주요 목적지 유지 여부를 실제 브라우저로 검증했습니다.

### v7 디자이너 리비전 — 동네 관찰 지도 + 골목 게시판

- 전담 UI 디자이너의 비평을 바탕으로 일반적인 청록 그라데이션·둥근 SaaS 카드·컬러 아이콘 타일·pill 배지를 제거했습니다.
- 미색 기록지, 먹색 정보, 흙빛 적갈색 행동색과 1px 도면선을 중심으로 한국 동네 관찰 도구의 고유한 시각 언어를 만들었습니다.
- 지도 상단을 떠 있는 유리 카드가 아닌 현장 계기판으로, 하단 시트를 주변 기록 게시판으로 재설계했습니다.
- 메뉴를 카드 모음 대신 선으로 구획된 `동네 생활판`으로 바꾸고 빠른 글쓰기를 목록 안의 `지금 이 동네에 기록 남기기` 행동으로 옮겼습니다.
- 커뮤니티의 건물·게시글을 그림자 카드 대신 촘촘한 기록 목록으로, 채팅을 말풍선 대신 시간 순 현장 로그로 정리했습니다.
- 하단 탐색은 활성 탭의 상단 적갈색 인덱스 선으로 현재 위치를 표시하며, 글쓰기 버튼은 지도와 커뮤니티에서만 문맥에 맞게 노출합니다.
- 320×568과 기본 브라우저 크기에서 지도·커뮤니티·채팅·메뉴를 다시 검증했습니다.
- `MenuPage` 회귀 테스트를 추가해 동네 둘러보기·정보와 안전·전용 기록 버튼이 테마 변경 뒤에도 유지되는지 검증합니다.

### v6 사용자 체감 UI 업그레이드

- 하단 탐색의 이모지를 일관된 선형 SVG 아이콘으로 교체하고 활성 탭을 배경·색상·레이블로 명확히 구분했습니다.
- 메뉴 홈을 브랜드 헤더와 `커뮤니티 둘러보기`·`설정과 보안` 그룹으로 재구성했습니다. 알림·복구 코드·위치 보호 기능은 유지하면서 탐색 순서를 단순화했습니다.
- 지도 헤더를 동네명과 검색 중심의 글래스 카드로 바꾸고, 필터·핀·내 위치 컨트롤과 글쓰기 버튼이 하단 시트 내용을 가리지 않도록 배치했습니다.
- 지도 시트에 주변 글 개수와 명확한 섹션 제목을 추가하고, 빈 동네 안내·첫 소식 CTA·커뮤니티 이동 버튼의 시각적 우선순위를 강화했습니다.
- 커뮤니티의 페이지 헤더·검색·필터·건물·게시글 카드를 하나의 표면·모서리·그림자 시스템으로 통일했습니다.
- 채팅에 실시간 연결 상태를 표시하고 메시지 영역과 입력창을 분리해 가독성을 높였습니다.
- 모바일에서는 큰 터치 영역과 안전 영역을 유지하고, 넓은 화면에서는 앱 폭을 제한해 콘텐츠가 과도하게 벌어지지 않게 했습니다.
- `TabBar` 회귀 테스트를 추가해 활성 탭의 접근성 상태와 탭 전환 동작을 검증합니다.

### 지도·위치·핀

- 위치 확인 또는 재시도 중에도 지도 DOM을 유지해 무한 로딩 가능성을 제거했습니다.
- 신뢰할 수 있는 현재 위치가 없으면 핀 생성을 차단합니다.
- 핀 생성 응답이 유실되어 재시도하더라도 같은 요청 ID와 소유 토큰을 재사용해 중복 핀을 방지합니다.
- 터치스크린 노트북에서도 마우스·트랙패드로 지도 위치를 선택할 수 있습니다.
- 장소검색 결과를 선택하면 마커뿐 아니라 장소 상세 카드도 즉시 열립니다.
- 장소검색 중·결과 개수를 표시하고 닫기 버튼, 바깥 클릭, Escape로 결과를 닫을 수 있습니다.
- 지도 탭이 숨겨지면 핀 정리·조회·실시간 구독을 중지하고 복귀할 때 다시 연결합니다.

### 게시글·커뮤니티

- 게시글 상세 화면에서 작성자가 바로 수정할 수 있습니다.
- 게시글 삭제 전에 확인·취소 단계를 표시하고, 키보드 포커스를 안전한 취소 버튼과 원래 버튼으로 복귀시킵니다.
- 모든 카테고리를 끈 경우 커뮤니티 화면에서 한 번에 다시 켤 수 있습니다.
- 카테고리 칩과 카드의 작은 라벨이 WCAG AA 명암 대비를 충족하도록 전경색을 분리했습니다.
- 글쓰기 모달이 부모 화면의 주기적 갱신 때마다 입력 포커스를 빼앗지 않도록 수정했습니다.
- 업로드가 지원되지 않는 HEIC 등의 형식은 미리보기 전에 안내하며, JPG·PNG·WebP·GIF만 선택할 수 있습니다.

### 채팅·접근성

- 사용자가 이전 메시지를 읽는 중에는 5초 갱신이 화면을 맨 아래로 강제 이동하지 않습니다.
- 브라우저별 스크롤 API 차이를 처리해 예약된 채팅 스크롤이 테스트와 구형 환경에서 실패하지 않습니다.
- 채팅 탭이 보일 때만 메시지 polling을 실행합니다.
- 장소 카드와 주요 다이얼로그에 키보드 포커스 순환, Escape 닫기, 포커스 복귀를 적용했습니다.
- 새 채팅과 댓글을 보조기기가 인식할 수 있도록 live region을 추가했습니다.
- 모션 축소 설정에서는 게시글·장소 카드 애니메이션도 비활성화됩니다.

### 2026-07-19 기능·보안 계약 일치 업그레이드

- 화면에 먼저 추가돼 있던 신고·댓글 답글·이모지 반응·복구 코드·푸시 설정을 `20260719_v4_feature_hardening.sql`로 재현 가능하게 만들었습니다.
- 소유 토큰은 SHA-256 hash로 확인하고, 복구용 사본은 기기 복구 코드로 AES-256 암호화해 plaintext로 저장하지 않습니다.
- 신고자·반응자·푸시 기기 식별값도 hash만 저장하고, 민감 테이블은 RLS와 권한 회수로 직접 접근을 차단합니다.
- 답글을 포함한 모든 댓글 쓰기를 `create_comment_v3` 원자 RPC로 통일했습니다.
- 5회 신고 자동 숨김, 중복 신고 방지, 허용 이모지 반응 토글, 최대 한 단계 답글을 DB 통합 테스트로 검증합니다.
- 숨김 글·댓글은 클라이언트 필터뿐 아니라 RLS 조회 정책에서도 제외합니다.

## 데이터베이스 적용

앱을 배포하기 전에 다음 migration을 순서대로 Supabase에 적용해야 합니다.

1. `supabase/migrations/20260718_v1_security_hardening.sql`
2. `supabase/migrations/20260719_v4_feature_hardening.sql`
3. `supabase/migrations/20260720024147_v6_auth_moderation_admin_security.sql`

세 migration은 다음을 포함합니다.

- 게시글·핀·댓글·채팅의 직접 쓰기 차단
- 작성자 소유권을 확인하는 원자 RPC
- 좋아요/현황 확인의 중복 방지와 원자 카운터
- 쓰기 빈도 제한
- 서버 반경 필터를 거친 채팅 조회
- 입력값·좌표 제약과 이미지 정리 대기열
- 암호화된 익명 소유권 복구
- 신고·댓글 반응·답글·푸시 설정의 hash 기반 RPC와 RLS
- 숨김 콘텐츠의 DB 조회 차단
- 로그인 프로필과 사용자명 RPC의 재현 가능한 정의
- 로그인 계정 기반 신고 중복 방지
- 비공개 관리자 역할, 신고 검토 큐, 원자 조치와 감사 로그
- Storage 객체 목록 열람 최소 권한 정책

기존 운영 데이터가 제약 조건을 위반하면 migration 검증 단계가 실패할 수 있으므로, 적용 전 백업과 staging 데이터 점검이 필요합니다. v1 hardening이 과거에 이미 적용되어 복구용 암호문 없이 plaintext 소유 토큰이 제거된 기존 행은 새 복구 코드로 되살릴 수 없으며, 기존 브라우저의 로컬 소유 토큰으로는 계속 관리할 수 있습니다. 실제 푸시 발송 worker·이미지 정리 worker·운영 모니터링은 PM 배포 단계에서 별도로 연결해야 합니다.
